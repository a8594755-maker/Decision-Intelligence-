# 資料上傳完整流程說明

## 概述

本系統提供完整的資料上傳、驗證、清洗和寫入流程，確保只有符合規格的資料才會被儲存到資料庫。

---

## 完整流程（6 個步驟）

### Step 1: 選擇上傳類型 📝

**目的**：告訴系統要上傳哪種類型的資料

**可選類型**：
- 📦 **收貨記錄 (Goods Receipt)** - 用於計算不良率和準時率
- 💰 **價格歷史 (Price History)** - 用於計算價格波動度
- 🏢 **供應商主檔 (Supplier Master)** - 創建或更新供應商資訊

**操作**：
1. 在下拉選單中選擇資料類型
2. 系統會顯示該類型的必填欄位說明

**注意事項**：
- 必須先選擇類型才能上傳檔案
- 不同類型有不同的欄位要求

---

### Step 2: 上傳檔案 📤

**目的**：上傳包含資料的 Excel 或 CSV 檔案

**支援格式**：
- `.xlsx` (Excel 2007+)
- `.xls` (Excel 97-2003)
- `.csv` (逗號分隔值)

**檔案要求**：
- 最大 10MB
- 第一行必須是欄位名稱
- 至少要有一筆資料

**操作**：
1. 點擊「選擇檔案上傳」按鈕
2. 選擇要上傳的檔案
3. 系統自動解析檔案內容

**範例 Excel 格式**：

```
| 供應商名稱 | 料號   | 收貨日期   | 收貨數量 | 拒收數量 |
|-----------|--------|-----------|---------|---------|
| 供應商A    | MAT001 | 2024-01-15| 100     | 5       |
| 供應商B    | MAT002 | 2024-01-16| 200     | 0       |
```

---

### Step 3: 欄位映射 🔗

**目的**：將 Excel 欄位對應到系統欄位

**操作**：
1. 系統顯示所有 Excel 欄位
2. 為每個 Excel 欄位選擇對應的系統欄位
3. 必填欄位必須完成映射（標示為紅色 `*`）

**映射範例**：

| Excel 欄位 | 系統欄位 |
|-----------|---------|
| 供應商名稱 | supplier_name ✓ |
| 料號 | material_code ✓ |
| 收貨日期 | actual_delivery_date ✓ |
| 收貨數量 | received_qty ✓ |
| 拒收數量 | rejected_qty |

**必填欄位檢查**：
- 系統會自動檢查所有必填欄位是否都有被映射
- 未映射的必填欄位會以紅色警告顯示
- 只有完成所有必填欄位映射才能進入下一步

**提示**：
- 欄位名稱不需要完全一樣，只要內容對應即可
- 選填欄位可以選擇「不映射」

---

### Step 4: 資料驗證與清洗 ✅

**目的**：檢查資料是否符合格式要求，並自動清洗轉換

**驗證內容**：

#### 1. 必填欄位檢查
- 檢查所有必填欄位是否有值
- 空值會被標記為錯誤

#### 2. 資料類型轉換

**數字欄位**：
- 自動移除逗號、空格、貨幣符號
- 轉換為純數字
- 範例：`"1,234.56"` → `1234.56`

**日期欄位**：
- 支援多種格式自動轉換
- 統一轉為 ISO 格式 (YYYY-MM-DD)
- 支援格式：
  - `2024-01-15` → `2024-01-15`
  - `2024/01/15` → `2024-01-15`
  - `15/01/2024` → `2024-01-15`
  - `20240115` → `2024-01-15`

**布林值欄位**：
- 自動識別：`true`, `yes`, `1`, `是` → `true`
- 自動識別：`false`, `no`, `0`, `否` → `false`

#### 3. 數值範圍檢查
- 數量必須 >= 0
- 價格必須 >= 0

#### 4. 預設值套用
- 選填欄位若為空，自動套用預設值
- 範例：`uom` 預設為 `"pcs"`

**驗證結果顯示**：

系統會顯示四個統計指標：
- 📊 **總行數**：上傳的總資料筆數
- ✅ **有效資料**：通過驗證的筆數
- ❌ **錯誤資料**：存在錯誤的筆數
- 📈 **成功率**：有效資料的百分比

**錯誤詳情表格**：

如果有錯誤資料，系統會顯示前 10 筆，包含：
- 行號
- 錯誤欄位名稱
- 原始值
- 詳細錯誤說明

範例錯誤：
```
第 3 行:
  - 收貨數量必須是數字，但得到：abc
  
第 5 行:
  - 供應商名稱為必填欄位，不可為空
  - 實際交貨日期的日期格式不正確：2024-13-45
```

---

### Step 5: 檢視結果並決定處理方式 🔍

**目的**：確認要寫入的資料

**三種情況**：

#### 情況 1: 全部資料有效 ✓
- 顯示綠色成功訊息
- 可以直接寫入全部資料

#### 情況 2: 部分資料有效 ⚠️
- 顯示藍色提示框
- 說明將儲存多少筆有效資料，略過多少筆錯誤資料
- **錯誤資料不會被寫入資料庫**

#### 情況 3: 全部資料錯誤 ❌
- 顯示黃色警告
- 「正式寫入資料庫」按鈕停用
- 必須返回修正資料後重新上傳

**重要提醒**：
> 💡 系統**只會寫入有效資料 (validRows)**，錯誤資料 (errorRows) 會被自動略過，不會寫入資料庫。

---

### Step 6: 正式寫入資料庫 💾

**目的**：將有效資料儲存到資料庫

**寫入流程**：

#### 收貨記錄 (Goods Receipt)
1. 保存原始檔案記錄到 `user_files` 表
2. 查詢或創建供應商（寫入 `suppliers` 表）
3. 查詢或創建物料（寫入 `materials` 表）
4. 批量插入收貨記錄到 `goods_receipts` 表

#### 價格歷史 (Price History)
1. 保存原始檔案記錄
2. 查詢或創建供應商
3. 查詢或創建物料
4. 批量插入價格記錄到 `price_history` 表

#### 供應商主檔 (Supplier Master)
1. 保存原始檔案記錄
2. 批量插入供應商到 `suppliers` 表

**寫入的資料包含**：
- `user_id`：記錄資料所屬的使用者
- `upload_file_id`：關聯到原始上傳檔案
- 所有清洗和轉換後的欄位資料

**成功訊息**：
- 如果全部有效：`成功儲存全部 100 筆資料`
- 如果部分有效：`成功儲存 97 筆有效資料，3 筆錯誤資料已略過`

**操作**：
1. 檢視驗證結果
2. 點擊「正式寫入資料庫」按鈕
3. 等待儲存完成（按鈕顯示「儲存中...」）
4. 看到成功訊息後，系統自動返回起始頁面

## 常見問題 FAQ

### Q1: 如果有錯誤資料怎麼辦？
**A**: 系統會自動略過錯誤資料，只寫入有效資料。錯誤資料不會影響有效資料的儲存。

### Q2: 錯誤資料會被刪除嗎？
**A**: 不會。錯誤資料只是不會被寫入資料庫，原始檔案仍會保存在 `user_files` 表中。

### Q3: 可以只修正錯誤資料重新上傳嗎？
**A**: 可以。修正 Excel 檔案中的錯誤資料後，重新執行上傳流程即可。

### Q4: 日期格式一定要是 YYYY-MM-DD 嗎？
**A**: 不用。系統支援多種日期格式，會自動轉換。但建議使用 YYYY-MM-DD 格式最保險。

### Q5: 如果供應商或物料已存在會怎樣？
**A**: 系統會自動查詢已存在的記錄並使用，不會重複創建。

### Q6: 上傳後可以修改資料嗎？
**A**: 資料寫入後需要透過其他功能進行修改。建議上傳前確認資料正確性。

### Q7: 成功率低於 50% 怎麼辦？
**A**: 系統會顯示警告訊息。建議檢查 Excel 資料是否符合格式要求，修正後重新上傳。

### Q8: 選填欄位一定要映射嗎？
**A**: 不用。選填欄位可以選擇「不映射」，系統會使用預設值或留空。

## 最佳實務建議

### 1. 上傳前準備
- ✅ 確認 Excel 第一行是欄位名稱
- ✅ 檢查日期格式（建議使用 YYYY-MM-DD）
- ✅ 數字欄位不要包含文字
- ✅ 必填欄位確保有值

### 2. 資料品質
- ✅ 統一日期格式
- ✅ 數字使用相同小數位數
- ✅ 供應商名稱保持一致（避免同一供應商有多種寫法）
- ✅ 料號格式統一

### 3. 分批上傳
- ✅ 大量資料建議分批上傳（每批 1000 筆以內）
- ✅ 先上傳小樣本測試
- ✅ 確認無誤後再上傳完整資料

### 4. 錯誤處理
- ✅ 記錄錯誤訊息
- ✅ 修正 Excel 原始檔案
- ✅ 重新上傳
- ✅ 不要手動修改資料庫

## 技術架構

### 前端流程
```
EnhancedExternalSystemsView.jsx
  ├─ Step 1: 類型選擇 (UPLOAD_SCHEMAS)
  ├─ Step 2: 檔案上傳 (XLSX.read)
  ├─ Step 3: 欄位映射 (columnMapping)
  ├─ Step 4: 驗證清洗 (validateAndCleanData)
  └─ Step 5-6: 寫入資料庫 (handleSave)
```

### 後端資料流
```
validRows
  ├─ user_files (原始檔案)
  ├─ suppliers (供應商)
  ├─ materials (物料)
  └─ goods_receipts / price_history (交易資料)
```

### 關鍵函數
- `validateAndCleanData()` - 完整驗證流程
- `handleSave()` - 資料庫寫入主函數
- `saveGoodsReceipts()` - 收貨記錄寫入
- `savePriceHistory()` - 價格歷史寫入
- `saveSuppliers()` - 供應商寫入

## 附錄：Operational Costs 上傳指引

> 原文件：OPERATIONAL_COSTS_UPLOAD_GUIDE.md（已合併）

### Step 1: Prepare Data File
- Use provided template: `templates/operational_costs.csv`
- Required fields:
  - `CostDate` (YYYY-MM-DD)
  - `DirectLaborHours`
  - `DirectLaborRate`
  - `ProductionOutput`
- Optional fields:
  - `IndirectLaborHours`
  - `IndirectLaborRate`
  - `MaterialCost`
  - `OverheadCost`
  - `Notes`

### Step 2: Upload Data
1. Go to Data Upload page
2. Select "Operational Costs" type
3. Drag and drop your CSV file
4. Map columns (automatic mapping works for template fields)
5. Review validation results
6. Click "Save Data"

### Step 3: Verify Upload
1. Go to Cost Analysis → Operational Cost tab
2. Check latest date appears
3. Verify cost per unit calculation

### Troubleshooting
- **Invalid date format**: Use YYYY-MM-DD
- **Negative cost per unit**: Check production output value
- **Missing data**: Ensure required fields are populated
